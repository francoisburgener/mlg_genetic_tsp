1. Minimal pyevolve Genetic Algorithm

The notebook GA_evo-bitstring-minimal.ipynb example uses pyevolve to evolve a bitstring of only ones.

2. Configuring the GA algorithm and plotting the evolution

The notebook GA_evo-bitstring.ipynb example uses pyevolve to evolve a bitstring of only ones. It allows the user to configure the GA by indicating the size of the population, the mutation and crossover parameters, etc. It moreover, records statistics about the evolved population to allow the user to plot the evolution of the fitness over generations.

3. Evolution of a list of values

The notebook GA_evo-list.ipynb example uses pyevolve to evolve a chromosome composed of a list of elements instead of a bitstring. A minimal application consists on evolving a list of only zeroes, starting from random integer values in a given range. This example introduces the use of the method stepCallback that allows the user to monitor the evolution process. Here, we print the current best solution found by the genetic algorithm.

4. Evolution of a string: Dawkin's Weasel experiment

The notebook GA_evo-string.ipynb example uses pyevolve to evolve a chromosome composed of a list of integers in the range 65 to 95, which includes the ASCII letters 'A' to 'Z' and '_', herein used as a space between words. We attempt to evolve the sequence of letters that compose the phrase: "METHINKS_IT_IS_LIKE_A_WEASEL".

5. Function maximization

The notebook GA_evo-binaryMaxFunc.ipynb example uses pyevolve to evolve the parameters of an equation (shown below) that we want to maximize:

f = (x0 * x1 * x2 * x3) / (x4 * x5 * x6 * x7), where x0, x1, ... x7 are integers in the range 1 < xi < 16.

The solution to this problem is very easy... f is maximized when x0 = x1 = x2 = x3 = 16 and x4 = x5 = x6 = x7 = 1.

The interest of this example is that it will show you how important is the coding of the solution into the efficacy of a genetic algorithm. To understand this, run the notebook and see if it can solve the proposed problem. Observe how we coded the values: every xi codes in binary an integer value, thus xi = bit3 * 2^3 + bit2 * 2^2 + bit1 * 2^1 + bit0 * 2^0, and '0000' codes for 16.

6. The Traveling Salesman Problem (TSP)

The travelling salesman problem (TSP) asks the following question: Given a list of cities and the distances between each pair of cities, what is the shortest possible route that visits each city exactly once and returns to the origin city? It is an NP-hard problem in combinatorial optimization, important in operations research and theoretical computer science [Wikipedia].

The objective of this exercise is to solve the TSP problem for a set of 14 cities in Burma, officially the Republic of the Union of Myanmar. The following vectors give the GPS position of each city. 

   LAT = [16.47, 16.47, 20.09, 22.39, 25.23, 22.00, 20.47, 
          17.20, 16.30, 14.05, 16.53, 21.52, 19.41, 20.09]

   LON = [96.10, 94.44, 92.54, 93.37, 97.24, 96.05, 97.02, 
          96.29, 97.38, 98.12, 97.38, 95.59, 97.13, 94.55]

Consider that the distance between two cities is not the Euclidean distance. You have to consider that they are points on the surface of the Earth, which can be considered to be an oblate spheroid.
